## 1. Математичні основи: згортка та теорема про згортку

### Просторова (spatial) згортка

Згортка — це фундаментальна операція у обробці сигналів і зображень. Для двовимірного дискретного зображення `f` розміром `H×W` та ядра (фільтра) `g` розміром `kH×kW` дискретна згортка визначається як:

```
(f * g)[i, j] = Σ_m Σ_n  f[i - m, j - n] · g[m, n]
```

Тобто для кожного пікселя вихідного зображення ми зсуваємо ядро, обчислюємо поелементні добутки між ядром та відповідною областю зображення і підсумовуємо їх.

**Обчислювальна складність:** якщо зображення має розмір `N×N`, а ядро `k×k`, то наївна реалізація має складність **O(N² · k²)**.

Для великих ядер (наприклад `31×31`) це стає дуже витратним. Тому і з'являється FFT-підхід.

### Теорема про згортку (Convolution Theorem)

Це один із найважливіших результатів у теорії сигналів:

> **Згортка у просторовій області = поелементний добуток у частотній області.**

Формально:

```
F{ f * g } = F{f} · F{g}
```

де `F{}` — перетворення Фур'є. Тобто:

```
f * g = F⁻¹{ F{f} · F{g} }
```

Замість дорогої просторової згортки можна:
1. Перейти у частотну область (FFT обох сигналів)
2. Поелементно перемножити
3. Повернутися у просторову область (IFFT)

Складність цього підходу: **O(N² · log N)** — значно краще для великих ядер.

## 2. Дискретне перетворення Фур'є (ДПФ) для зображень

### Одновимірне ДПФ

Для послідовності `x[n]`, де `n = 0, 1, ..., N-1`:

```
X[k] = Σ_{n=0}^{N-1}  x[n] · e^{-j·2π·k·n/N}
```

де `j = √(-1)`, `k` — частотний індекс, `X[k]` — комплексне число, що кодує амплітуду і фазу гармоніки з частотою `k/N`.

**Зворотне ДПФ (ОДПФ):**

```
x[n] = (1/N) · Σ_{k=0}^{N-1}  X[k] · e^{j·2π·k·n/N}
```

### Двовимірне ДПФ (для зображень)

Для зображення `f[m, n]` розміром `M×N`:

```
F[u, v] = Σ_m Σ_n  f[m, n] · e^{-j·2π·(u·m/M + v·n/N)}
```

Геометрично: `u` та `v` — просторові частоти по вертикалі та горизонталі відповідно. Центр спектра (`u=0, v=0`) відповідає DC-компоненті (середньому значенню пікселів). Краї — високі частоти (дрібні деталі, краї).

### Кругова (circular) vs лінійна згортка

Ключова пастка: ДПФ за природою передбачає **кругову** згортку — сигнал вважається **циклічним** (кінець "зшивається" з початком). Натомість у CNN нам потрібна **лінійна** згортка (без циклічних артефактів на краях).

**Розв'язок:** перед FFT зображення **доповнюється нулями (zero-padding)** до розміру принаймні `(H + kH - 1) × (W + kW - 1)`. Це гарантує, що кругова згортка збіжиться з лінійною.

## 3. Швидке перетворення Фур'є (FFT) та його складність

### Алгоритм Кулі-Тьюкі (Cooley-Tukey)

Наївний DFT має складність O(N²). FFT — алгоритм, що рекурсивно розбиває задачу навпіл (divide and conquer), використовуючи симетрію коренів з одиниці. Для N, що є степенем двійки:

```
Складність FFT: O(N · log₂N)
```

Саме тому в коді є функція `_next_power_of_two(n)` — розміри дополнюються до найближчого степеня двійки для максимальної ефективності FFT.

```python
def _next_power_of_two(n: int) -> int:
    return 1 << math.ceil(math.log2(n))
```

`1 << k` — це `2^k` у бітовому зсуві. Наприклад, для `n=70` → `math.ceil(log2(70)) = 7` → `1 << 7 = 128`.

### rfft2 vs fft2

`torch.fft.rfft2` (real FFT) — спеціалізована функція для **дійсних** вхідних даних (зображення завжди дійсні числа). Завдяки симетрії Ерміта вихід має розмір `(H, W//2 + 1)` замість `(H, W)` — вдвічі менше пам'яті та обчислень. `irfft2` — відповідне зворотне перетворення.

## 4. Архітектура ResNet18

### Проблема деградації та Residual блоки

До ResNet (2015) глибокі мережі страждали від **деградації**: додавання більше шарів погіршувало точність навіть на тренувальних даних — не через перенавчання, а через труднощі оптимізації.

ResNet вирішує це **shortcut connections (skip connections)**:

```
y = F(x, {Wᵢ}) + x
```

Замість того, щоб шар вчився довільному перетворенню `H(x)`, він вчить **залишок** `F(x) = H(x) - x`. Якщо оптимальне перетворення близьке до тотожного, досить занулити ваги — набагато простіше для градієнтного спуску.

### Структура ResNet18

```
conv1       → 7×7, 64 канали, stride=2, padding=3
bn1 + relu
maxpool     → 3×3, stride=2

layer1      → 2 BasicBlock (64 → 64)
layer2      → 2 BasicBlock (64 → 128, перший з stride=2)
layer3      → 2 BasicBlock (128 → 256, перший з stride=2)
layer4      → 2 BasicBlock (256 → 512, перший з stride=2)

avgpool     → adaptive, вихід 1×1
fc          → Linear(512, num_classes)
```

**BasicBlock** (для ResNet18/34):
```
Conv2d(3×3) → BN → ReLU → Conv2d(3×3) → BN → (+x) → ReLU
```

"18" у назві — це кількість шарів з вагами (16 conv + 2 fc-еквіваленти).

### Чому саме conv1 та layer1 замінюємо на FFT?

- `conv1` — перший шар, 7×7 ядро, найбільший ефект від FFT при великих ядрах
- `layer1` — найраніший шар з найбільшими просторовими картами ознак (feature maps 64×64 при вході 64×64), де заміна найбільш помітна
- Глибокі шари (`layer3`, `layer4`) працюють з малими картами (8×8, 4×4) — там FFT не дає виграшу

## 5. Transfer learning: що, чому і як

### Концепція

**Transfer learning** — техніка, коли модель, натренована на одній задачі (зазвичай великому датасеті, як ImageNet), використовується як відправна точка для іншої задачі.

**Чому це працює?** Ранні шари CNN вчать универсальні риси: детектори країв, кутів, текстур. Ці риси корисні для будь-якого датасету зображень. Лише останні шари (голова) специфічні для конкретного завдання.

### Заморожування backbone

```python
def _freeze(module: nn.Module) -> None:
    for p in module.parameters():
        p.requires_grad_(False)
```

`requires_grad_(False)` — сказати PyTorch не обчислювати градієнти для цих параметрів під час backward pass. Це дає два ефекти:
1. **Ваги не змінюються** — backbone залишається "замороженим"
2. **Прискорення та економія пам'яті** — не треба зберігати графи обчислень для цих шарів

### Навчальна голова (head)

```python
backbone.fc = nn.Sequential(
    nn.Dropout(0.3),
    nn.Linear(in_features, num_classes),
)
```

Замінюємо оригінальний `fc` шар ResNet18 (розрахований на 1000 класів ImageNet) на новий лінійний шар для 10 класів CIFAR-10. Dropout(0.3) — регуляризація для запобігання перенавчанню: під час навчання 30% нейронів випадково "вимикаються" на кожному forward pass.

## 6. Файл `fft_conv.py` — детальний розбір

### Клас `FFTConv2d`

Це реалізація згорткового шару, який виконує згортку через добуток у просторі ДПФ. Ключова відмінність від `nn.Conv2d`: ваги фільтрів зберігаються як **buffer**, а не **Parameter**.

#### Різниця між `register_buffer` та `nn.Parameter`

| | `nn.Parameter` | `register_buffer` |
|---|---|---|
| Обчислення градієнтів | Так | Ні |
| Оновлення оптимізатором | Так | Ні |
| Збереження у state_dict | Так | Так |
| Переміщення на GPU (`model.cuda()`) | Так | Так |

```python
self.register_buffer("weight", weight)
```

Буфер "подорожує" з моделлю (GPU/CPU, state_dict), але **ніколи не навчається**. Це і є "банк фільтрів, що не змінюється".

#### Ініціалізація Kaiming (He initialization)

```python
nn.init.kaiming_uniform_(weight, a=math.sqrt(5))
```

Запропонована Каімінг Хе (автором ResNet) ініціалізація для шарів з ReLU-активацією. Варіанс ваг масштабується як `2/fan_in`, що запобігає вибуху/затуханню градієнтів у глибоких мережах. `a=math.sqrt(5)` — параметр для LeakyReLU-подібної активації (використовується в PyTorch за замовчуванням).

#### Метод `from_conv2d`

```python
@classmethod
def from_conv2d(cls, conv: nn.Conv2d) -> "FFTConv2d":
    layer = cls(...)
    with torch.no_grad():
        layer.weight.copy_(conv.weight.data)
```

Копіює **претреновані ваги** з `nn.Conv2d` у наш буфер. `torch.no_grad()` — контекстний менеджер, що вимикає обчислення графа автодиференціювання (економія пам'яті при операціях копіювання).

#### Метод `forward` — покроково

```python
def forward(self, x: torch.Tensor) -> torch.Tensor:
    B, C, H, W = x.shape
    kH, kW = self.kernel_size
```

**Крок 1: Zero-padding для лінійної згортки**

```python
pad_h = kH - 1
pad_w = kW - 1
x_pad = F.pad(x, (pad_w // 2, pad_w - pad_w // 2,
                   pad_h // 2, pad_h - pad_h // 2))
```

Доповнюємо зображення нулями на `kH-1` рядків та `kW-1` стовпців. Аргументи `F.pad` мають порядок `(left, right, top, bottom)`. Ділимо padding порівну між двома сторонами (або +1 на праву/нижню при непарному розмірі).

**Навіщо саме `kH - 1`?** При лінійній згортці двох послідовностей довжин `N` та `M` результат має довжину `N + M - 1`. Щоб кругова ДПФ-згортка збіглася з лінійною, потрібно доповнити до розміру не менше `N + M - 1`, тобто додати `M - 1 = k - 1` нулів.

**Крок 2: Вибір розміру FFT**

```python
fft_h = _next_power_of_two(x_pad.shape[-2])
fft_w = _next_power_of_two(x_pad.shape[-1])
```

Округлення до степеня двійки прискорює FFT-алгоритм Кулі-Тьюкі.

**Крок 3: FFT зображення**

```python
X = torch.fft.rfft2(x_pad, s=(fft_h, fft_w))
```

`rfft2` — 2D дійсне FFT. Параметр `s=(fft_h, fft_w)` вказує розмір FFT (зображення неявно доповнюється нулями до цього розміру). Результат `X` має форму `(B, C, fft_h, fft_w//2+1)` — комплексний тензор.

**Крок 4: FFT ядра**

```python
W_freq = torch.fft.rfft2(self.weight, s=(fft_h, fft_w))
```

Ядро також перетворюється у частотний простір того ж розміру. Форма: `(out_ch, in_ch/g, fft_h, fft_w//2+1)`.

**Крок 5: Групова згортка у частотному просторі**

```python
for grp in range(g):
    x_grp = X[:, grp * in_per_g:(grp + 1) * in_per_g]
    w_grp = W_freq[grp * out_per_g:(grp + 1) * out_per_g]
    y_grp = torch.einsum("bihw,oihw->bohw", x_grp, w_grp)
    out_chunks.append(y_grp)
Y = torch.cat(out_chunks, dim=1)
```

**Групова згортка** — розбиває канали на `g` незалежних груп (використовується в ResNet для depthwise-подібних операцій). При `groups=1` (стандартний випадок) цикл виконується один раз.

**`torch.einsum("bihw,oihw->bohw", x_grp, w_grp)`** — нотація Ейнштейна:
- `b` — batch
- `i` — вхідні канали
- `o` — вихідні канали
- `h, w` — просторові частоти

Множимо кожен вхідний канал на відповідне ядро і підсумовуємо по `i`. Це і є поелементне множення у частотному просторі = згортка у просторовому.

**Крок 6: Зворотне FFT та обрізання**

```python
out = torch.fft.irfft2(Y, s=(fft_h, fft_w))
out = out[:, :, :H, :W]
```

Повертаємося у просторовий простір. Обрізаємо до вихідного розміру `H×W` (padding, доданий на кроці 1, "поглинув" крайові ефекти).

**Крок 7: Зміщення (bias)**

```python
if self.bias is not None:
    out = out + self.bias.view(1, -1, 1, 1)
```

Bias — єдиний **навчальний параметр** у цьому шарі. `view(1, -1, 1, 1)` розширює тензор форми `(out_ch,)` до `(1, out_ch, 1, 1)` для broadcast-сумування з тензором `(B, out_ch, H, W)`.

## 7. Файл `models.py` — детальний розбір

### `build_model_a` — базова модель

```python
backbone = models.resnet18(weights=models.ResNet18_Weights.DEFAULT)
_freeze(backbone)
backbone.fc = nn.Sequential(
    nn.Dropout(0.3),
    nn.Linear(in_features, num_classes),
)
```

1. Завантажуємо ResNet18 з вагами ImageNet (`DEFAULT` = найкращі доступні ваги)
2. Заморожуємо всі параметри backbone
3. Замінюємо fc шар — тепер навчаються лише `512 * 10 + 10 = 5130` параметрів

### `build_model_b` — модель з FFT-банком фільтрів

```python
backbone.conv1 = FFTConv2d.from_conv2d(backbone.conv1)

for name, module in backbone.layer1.named_modules():
    if isinstance(module, nn.Conv2d):
        parent = backbone.layer1
        parts = name.split(".")
        for part in parts[:-1]:
            parent = getattr(parent, part)
        fft_layer = FFTConv2d.from_conv2d(module)
        setattr(parent, parts[-1], fft_layer)
```

**Замінюємо `conv1`:** перший шар ResNet18 — `Conv2d(3, 64, 7, stride=2, padding=3)`. Це ядро 7×7, тому FFT-підхід тут особливо доречний.

**Ітерація по `layer1`:** `named_modules()` повертає всі модулі рекурсивно у форматі `(ім'я, модуль)`. Для кожного знайденого `nn.Conv2d` ми:
1. Розбиваємо ім'я по `.` (наприклад `"0.conv1"` → `["0", "conv1"]`)
2. Навігуємо до батьківського модуля через `getattr`
3. Замінюємо атрибут на `FFTConv2d` через `setattr`

Зауважте: і `model_a`, і `model_b` мають **однакову кількість навчальних параметрів** (5130) — ваги FFTConv2d є буферами, не параметрами.

### Секція 1: Налаштування середовища

```python
SEED = 42
DEVICE = torch.device('cuda' if torch.cuda.is_available() else 'cpu')

def set_seed(seed=SEED):
    torch.manual_seed(seed)
    torch.cuda.manual_seed_all(seed)
    np.random.seed(seed)
```

**Фіксація random seed** — критична вимога задання. Гарантує відтворюваність результатів: при однаковому seed обидві моделі отримують однаково ініціалізовані головні шари, однакову послідовність батчів, однакові dropout-маски.

`torch.cuda.manual_seed_all(seed)` — необхідний при використанні кількох GPU або для детермінізму CUDA-операцій.

### Секція 2: Датасет CIFAR-10

CIFAR-10 — 60 000 кольорових зображень 32×32 пікселів у 10 класах: літак, автомобіль, птах, кіт, олень, собака, жаба, кінь, корабель, вантажівка. 50 000 тренувальних і 10 000 тестових.

```python
train_tf = transforms.Compose([
    transforms.Resize(64),
    transforms.RandomHorizontalFlip(),
    transforms.RandomCrop(64, padding=4),
    transforms.ToTensor(),
    transforms.Normalize([0.4914, 0.4822, 0.4465],
                         [0.2470, 0.2435, 0.2616]),
])
```

**Resize(64):** ResNet18 розрахований на більші зображення (224×224). Масштабуємо до 64×64 — компроміс між якістю та швидкістю.

**RandomHorizontalFlip:** data augmentation — горизонтальне відображення з ймовірністю 0.5. Подвоює ефективний розмір датасету.

**RandomCrop(64, padding=4):** спочатку доповнюємо по 4 пікселі з кожного боку, потім вирізаємо случайну ділянку 64×64. Навчає модель бути стійкою до малих зсувів об'єкта.

**Normalize:** нормалізація кожного кольорового каналу (R, G, B). Значення `[0.4914, 0.4822, 0.4465]` — середні значення пікселів CIFAR-10, `[0.247, 0.243, 0.261]` — стандартні відхилення. Нормалізація прискорює збіжність оптимізатора.

**ToTensor:** конвертує PIL-зображення `(H, W, C)` [0..255] у тензор `(C, H, W)` [0..1].

### Секція 3: Цикл навчання

```python
def train_one_epoch(model, loader, criterion, optimizer, device):
    model.train()
    for x, y in loader:
        optimizer.zero_grad()
        logits = model(x)
        loss = criterion(logits, y)
        loss.backward()
        optimizer.step()
```

**`model.train()`** — переводить модель у режим навчання: активує Dropout та BatchNorm з поточними статистиками батчу (а не накопиченими).

**`optimizer.zero_grad()`** — обнуляє накопичені градієнти. PyTorch за замовчуванням **накопичує** градієнти (accumulates), тому їх треба очищати на початку кожного кроку.

**`loss.backward()`** — зворотне поширення (backpropagation). Обчислює градієнти функції втрат відносно всіх параметрів з `requires_grad=True` через ланцюгове правило диференціювання.

**`optimizer.step()`** — оновлює параметри по формулі Adam:
```
m_t = β₁ · m_{t-1} + (1 - β₁) · g_t
v_t = β₂ · v_{t-1} + (1 - β₂) · g_t²
θ_t = θ_{t-1} - α · m̂_t / (√v̂_t + ε)
```
де `g_t` — градієнт, `m_t` — перший момент (аналог імпульсу), `v_t` — другий момент (адаптивне масштабування кроку).

**`CrossEntropyLoss`** — для класифікації. Внутрішньо є `LogSoftmax + NLLLoss`:
```
L = -log( exp(logit_y) / Σ exp(logit_k) )
```

**`CosineAnnealingLR`** — планувальник learning rate за косинусним графіком:
```
lr_t = lr_min + 0.5 · (lr_max - lr_min) · (1 + cos(π · t / T_max))
```
Починає з `lr=1e-3`, плавно зменшує до нуля за `T_max=epochs` кроків. Допомагає точніше сходитися до мінімуму наприкінці.

### Секція 4: Збереження найкращої моделі

```python
if val_acc > best_acc:
    best_acc = val_acc
    best_state = copy.deepcopy(model.state_dict())
```

**`state_dict()`** — словник усіх параметрів та буферів моделі. **`copy.deepcopy`** — повна незалежна копія (без deepcopy зберігалося б посилання, і `best_state` змінювався б разом з моделлю).

## 9. Метрики: ROC-AUC, F1, MAPE

### ROC-AUC

**ROC (Receiver Operating Characteristic) крива** — графік TPR (True Positive Rate = чутливість) проти FPR (False Positive Rate = 1 - специфічність) при різних порогах класифікації.

**AUC (Area Under Curve)** — площа під ROC-кривою. Значення від 0.5 (випадковий класифікатор) до 1.0 (ідеальний).

Для багатокласової задачі (10 класів) використовується **OvR (One vs Rest)**: для кожного класу будується окрема бінарна ROC-крива (поточний клас проти всіх інших), потім обчислюється макро-середнє.

```python
roc_auc = roc_auc_score(labels_bin, probs, multi_class='ovr', average='macro')
```

`label_binarize` перетворює мітки `[0, 2, 5, ...]` у матрицю one-hot `[[1,0,0,...], [0,0,1,...], ...]`.

### F1-score

```
Precision = TP / (TP + FP)    # з усього що ми передбачили як клас X, скільки справді X
Recall    = TP / (TP + FN)    # зі всіх справжніх X, скільки ми знайшли
F1        = 2 · (P · R) / (P + R)    # гармонічне середнє
```

**Weighted F1** — зважує F1 кожного класу пропорційно до кількості його прикладів у тестовому наборі. Підходить для збалансованих датасетів (CIFAR-10 має по 1000 прикладів на клас).

```python
f1 = f1_score(labels, preds, average='weighted')
```

### MAPE (Mean Absolute Percentage Error)

MAPE — класична метрика для регресії, але тут застосована нестандартно: порівнюються передбачені **ймовірності** (softmax outputs) проти **one-hot векторів** (цільових "ймовірностей"):

```
MAPE = mean( |target - predicted| / |target| ) × 100%
```

```python
mask = one_hot > 0   # рахуємо лише де target=1 (уникаємо ділення на 0)
mape = np.mean(np.abs(one_hot[mask] - probs[mask]) / (one_hot[mask] + 1e-8)) * 100
```

`1e-8` — числова стабільність (epsilon). Оскільки `one_hot[mask]` завжди 1.0, це спрощується до `mean(|1 - P(true_class)|) × 100%` — середнє відхилення передбаченої ймовірності правильного класу від 1.

### Фіксований seed при оцінці

```python
def compute_metrics(model, loader, name):
    set_seed()  # фіксуємо перед evaluate
    ...
```

Потрібен для того, щоб будь-які стохастичні операції (наприклад Dropout, якщо він активний в eval — але зазвичай `model.eval()` його вимикає) були однаковими для обох моделей.

## 10. Бенчмарк швидкості: малі vs великі фільтри

### Теоретичне обґрунтування

| Підхід | Складність | Переважає при |
|---|---|---|
| Просторова згортка | O(N² · k²) | малих ядрах (k << N) |
| FFT-згортка | O(N² · log N) | великих ядрах (k → N) |

**Точка перетину** (crossover point) — розмір ядра `k*`, при якому обидва підходи займають однаковий час. Залежить від розміру зображення, кількості каналів та апаратного забезпечення.

```python
kernel_sizes = [3, 5, 7, 11, 15, 21, 31]
```

- `k ∈ {3, 5}` — просторова згортка майже завжди швидша (CUDA-оптимізовані реалізації cuDNN)
- `k ∈ {15, 21, 31}` — FFT починає виграти або рівнятися

### Функція benchmark

```python
def bench_conv(layer, x, n_iter=50):
    # warm-up
    for _ in range(5):
        _ = layer(x)
    torch.cuda.synchronize()   # чекаємо завершення GPU операцій
    t0 = time.perf_counter()
    for _ in range(n_iter):
        _ = layer(x)
    torch.cuda.synchronize()
    return (time.perf_counter() - t0) / n_iter * 1000
```

**`torch.cuda.synchronize()`** — CUDA операції асинхронні: CPU продовжує виконання, не чекаючи GPU. `synchronize()` блокує CPU до завершення всіх CUDA-операцій. Без нього вимірювання часу було б некоректним (вимірювали б час постановки задачі в чергу, а не виконання).

**Warm-up (5 ітерацій)** — перші запуски можуть бути повільнішими через JIT-компіляцію, завантаження даних у кеш GPU, ініціалізацію cuBLAS. Warm-up виключає цей "холодний старт" з вимірювань.

## 11. Питання для самоперевірки

1. **Чому FFT-згортка еквівалентна просторовій?** — Теорема про згортку: `F{f*g} = F{f}·F{g}`. Добуток у частотному просторі відповідає згортці у просторовому.

2. **Що таке кругова згортка і чому вона проблематична?** — ДПФ передбачає циклічну природу сигналу. При згортці без padding кінець зображення "зшивається" з початком, що дає артефакти. Zero-padding розміром `k-1` усуває це.

3. **Навіщо `_next_power_of_two`?** — Алгоритм FFT Кулі-Тьюкі має оптимальну ефективність при довжинах, що є степенями двійки.

4. **Чим буфер відрізняється від Parameter у PyTorch?** — Parameter навчається (має градієнти, оновлюється оптимізатором). Буфер зберігається у state_dict і переміщується між пристроями, але не навчається.

5. **Що таке Kaiming initialization і навіщо вона?** — Ініціалізація ваг з варіансом `2/fan_in`. Запобігає вибуху/затуханню градієнтів у глибоких мережах з ReLU-активаціями.

6. **Чому model_a і model_b мають однакову кількість навчальних параметрів?** — Ваги FFTConv2d — буфери, не параметри. Обидві моделі навчають лише голову: `512×10 + 10 = 5130` параметрів.

7. **Що таке skip connections у ResNet?** — Додавання вхідного тензора до виходу блоку (`y = F(x) + x`). Вирішує проблему деградації, дозволяє вчити "залишок" замість повного перетворення.

8. **Що таке OvR (One vs Rest) для ROC-AUC?** — Для кожного з 10 класів будується бінарна задача класифікації. Обчислюється AUC окремо, потім макро-усереднюється.

9. **Навіщо `torch.cuda.synchronize()` у бенчмарку?** — CUDA асинхронна. Без синхронізації `time.perf_counter()` вимірює час CPU, а не час виконання GPU.

10. **За яких умов FFT-згортка швидша за просторову?** — При великих ядрах (k >> √(N log N)). Для k=3 просторова швидша (cuDNN-оптимізація); для k=31 FFT виграє.
