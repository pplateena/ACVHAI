# Лабораторна робота 1 — Висновки та спостереження

## Детектування ґрунтової площини
- HSV-маска ефективніша за RGB: колір фону в HSV компактний, легко відокремлюється від об'єктів.
- Морфологічні операції (erode + open) усувають шум на межах маски.

## Оператор Собеля
- Собель по кожному BGR-каналу окремо з вибором максимуму — краще, ніж сірий: дозволяє знайти краї між кольорами схожої яскравості (наприклад, жовтий на рожевому).
- Кут градієнта `arctan2(sy, sx)` визначає орієнтацію краю: вертикальні краї мають горизонтальний градієнт (`|sin(angle)| < thresh`).

## Класифікація пікселів
- Чотири класи: ґрунт, вертикальний край, горизонтальний край, грань.
- Контактний край (горизонтальний + ґрунт знизу) — жорстко фіксує Y=0 внизу об'єкта.
- Горизонтальний некontактний край: глибина постійна вздовж напрямку краю, тому похідна Y в перпендикулярному напрямку = 0.

## Лінійна система та розв'язання
- Система рідкісна (COO → CSR), розв'язується через `scipy.sparse.linalg.lsqr`.
- Вертикальні краї складають ~0.5% від усіх обмежень — вони заглушуються обмеженнями гладкості граней. Коефіцієнт VERT_WEIGHT необхідний для компенсації.
- Розв'язувач дає зміщення по Y: нормалізація через `Y -= Y[ground].mean()` повертає ґрунт до нуля.

## Відновлення Z
- Формула проекції: `y_math = Y·cos(φ) - Z·sin(φ)`, звідси `Z = (Y·cos(φ) + py - height/2) / sin(φ)`.
- Без інверсії Z: близькі пікселі — позитивні, далекі — від'ємні. Інверсія `Z = Z_raw.max() - Z_raw` виправляє орієнтацію.

## Головне обмеження підходу
- Пікселі граней (~99% обмежень) мають лише умови гладкості — не мають абсолютного якоря по висоті, тому розв'язувач тягне їх до Y≈0.
- Вертикальні краї дають правильний градієнт висоти, але інформація не поширюється в глиб граней через брак зв'язку між ними.
- Геометричний підхід (без розв'язувача) дає правильну Y для бічних граней, але всі пікселі грані опиняються на одній глибині Z — блок виглядає плоским.

## Практичний висновок
- Метод "blocks world" працює коректно для сцен з прямими кутами та достатньою кількістю вертикальних країв.
- Для даної сцени (кольорові куби на сірій підлозі) класифікація пікселів та відновлення Z виконуються правильно; основна проблема — недостатня щільність вертикальних країв для повноцінного відновлення висоти граней.
