# Алгоритм відновлення 3D з 2D зображення — покроково

## Крок 1 — Завантаження та підготовка зображення

Зображення завантажується та масштабується до фіксованого розміру 800×600. Це спрощує налаштування порогів, які залежать від розміру.

---

## Крок 2 — Виділення ґрунтової площини (HSV-маска)

**Мета:** визначити, які пікселі належать підлозі (Y=0), а які — об'єктам.

**Як:** конвертуємо зображення з BGR у HSV і задаємо діапазон кольорів фону через `cv2.inRange`. Інвертуємо маску — отримуємо маску переднього плану. Застосовуємо морфологічні операції:
- `MORPH_OPEN` — прибирає дрібні острівці шуму
- `erode` — стискає межі об'єктів, відокремлюючи їх від фону

Результат: `ground[py, px] = True` якщо піксель — це підлога або фон.

---

## Крок 3 — Градієнти Собеля

**Мета:** знайти краї між гранями об'єктів.

**Як:** застосовуємо оператор Собеля окремо по кожному з трьох каналів (B, G, R). Для кожного пікселя вибираємо канал із найсильнішим градієнтом. Це дозволяє знаходити краї між кольорами схожої яскравості, які сірий Собель пропустив би.

З компонент градієнта `(sx, sy)` рахуємо:
- `mag = sqrt(sx² + sy²)` — сила краю
- `angle = arctan2(sy, sx)` — напрямок градієнта (перпендикулярно до краю)

---

## Крок 4 — Класифікація пікселів

Кожен піксель отримує один із чотирьох класів:

| Клас | Умова | Значення |
|------|-------|----------|
| **ground** | HSV-маска | підлога, Y=0 |
| **vert_edge** | `mag > threshold` і `\|sin(angle)\| < thresh` | вертикальний край між гранями |
| **horiz_edge** | `mag > threshold` і не vert_edge | горизонтальний край (верх/низ об'єкта) |
| **face** | не ground і не edge | поверхня грані |

**Вертикальний край:** його нормаль вказує горизонтально → кут градієнта близький до 0° або 180° → `|sin(angle)|` малий.

**Контактний край** — особливий випадок horiz_edge: якщо знизу є ґрунт, а зверху немає — це нижня межа об'єкта, Y=0.

---

## Крок 5 — Побудова розрідженої системи обмежень

**Мета:** скласти лінійну систему `A · Y_vec = b`, де `Y_vec[x + y·width]` — висота кожного пікселя.

Матриця A будується у форматі COO (списки рядків, стовпців, значень), потім конвертується у CSR для ефективного розв'язання.

### Обмеження по класах:

**Ground** → `Y(x,y) = 0`
```
A[row, flat(x,y)] = 1,   b[row] = 0
```

**Vertical edge** → висота змінюється з глибиною зі швидкістю `1/cos(φ)`. Оскільки піксельна вісь Y напрямлена вниз (протилежно математичній), знак від'ємний:
```
Y(x,y+1) - Y(x,y-1) = -1/cos(φ)
```

**Contact edge** → `Y(x,y) = 0` (нижній край об'єкта на підлозі)

**Horizontal edge** (не контактний) → глибина постійна вздовж краю. Похідна Y у напрямку краю = 0:
```
-ny·(Y(x+1,y) - Y(x-1,y)) + nx·(Y(x,y+1) - Y(x,y-1)) = 0
```
де `(nx, ny)` — нормаль до краю (з градієнта Собеля).

**Face** → поверхня плоска, всі другі похідні = 0:
```
Y(x+2,y) - 2·Y(x,y) + Y(x-2,y) = 0   (d²Y/dx²)
Y(x,y+2) - 2·Y(x,y) + Y(x,y-2) = 0   (d²Y/dy²)
Y(x+1,y+1) - Y(x-1,y+1) - Y(x+1,y-1) + Y(x-1,y-1) = 0   (d²Y/dxdy)
```

### Посилення вертикальних країв:
Рядки вертикальних країв множаться на `VERT_WEIGHT`, бо їх ~0.5% від усіх обмежень — без підсилення вони заглушуються обмеженнями гладкості граней.

## Крок 6 — Розв'язання системи

```python
Y_flat = lsqr(A, b)[0]
```

`lsqr` — ітераційний метод найменших квадратів для розріджених систем. Система перевизначена (обмежень більше, ніж невідомих), тому знаходиться наближений розв'язок із мінімальною нормою нев'язки.

**Нормалізація:** розв'язувач не знає абсолютного нуля — зміщує все поле Y довільно. Виправляємо:
```python
Y = Y - Y[ground].mean()   # повертаємо ґрунт до нуля
Y = clip(Y, 0, None)       # прибираємо від'ємні залишки
```

## Крок 7 — Відновлення Z (глибини)

**Рівняння проекції** (з лекції):
```
y_math = Y·cos(φ) - Z·sin(φ)
```
де `y_math = height/2 - py` (математична вісь Y направлена вгору, піксельна — вниз).

**Звідси Z:**
```
Z = (Y·cos(φ) + py - height/2) / sin(φ)
```

Сирий Z: ближні пікселі → більше значення, далекі → менше. Інвертуємо:
```
Z = Z_raw.max() - Z_raw
```
Результат: ближнє = 0, дальнє = велике позитивне число.

**X** — просто піксельна колонка, зміщена до центру:
```
X = px - width/2
```

## Крок 8 — 3D-візуалізація

Піксели відображаються як хмара точок у Plotly Scatter3d:
- вісь X → ліво/право
- вісь Y → глибина (від камери вглиб)
- вісь Z → висота над ґрунтом

Колір кожної точки береться безпосередньо з оригінального зображення.
