# ЛР №2 — Перетворення Фур'є: детальне пояснення

---

## Загальна ідея лабораторної роботи

Будь-яке зображення можна розглядати не лише як набір пікселів у просторовій області, а й як суперпозицію синусоїд різних частот, амплітуд і фаз — у **частотній області**. Дискретне перетворення Фур'є (ДПФ) є математичним інструментом, що дозволяє перейти між цими двома представленнями.

Мета лабораторної — на практиці відчути, що несе амплітуда ДПФ, що несе фаза, як маніпуляції у частотній області впливають на просторове зображення, і як ДПФ можна використовувати замість прямої згортки для фільтрації.

---

## Математична основа

### Двовимірне ДПФ

Для зображення $f[m, n]$ розміром $M \times N$ ДПФ визначається як:

$$F[k, l] = \sum_{m=0}^{M-1} \sum_{n=0}^{N-1} f[m,n] \cdot e^{-j2\pi\left(\frac{km}{M} + \frac{ln}{N}\right)}$$

Результат $F[k, l]$ — **комплексне число** для кожної частоти $(k, l)$:
- **Амплітуда**: $|F[k, l]|$ — наскільки сильно ця частота представлена у зображенні
- **Фаза**: $\angle F[k, l] = \arg(F[k, l])$ — де у просторі розташований цей частотний компонент (зсув)

### Зворотне ДПФ (ЗЗДПФ)

$$f[m, n] = \frac{1}{MN} \sum_{k=0}^{M-1} \sum_{l=0}^{N-1} F[k,l] \cdot e^{j2\pi\left(\frac{km}{M} + \frac{ln}{N}\right)}$$

### fftshift

Стандартний `fft2` розміщує нульову частоту (DC) у кутах масиву. Щоб DC-компонента була у центрі (зручно для візуалізації і для кільцевих масок), застосовується `fftshift`. Перед `ifft2` потрібно застосувати `ifftshift` — щоб повернути порядок назад.

### Теорема про згортку

Якщо $g = f * h$ (просторова згортка), то у частотній області:

$$G[k,l] = F[k,l] \cdot H[k,l]$$

Це означає, що фільтрувати зображення можна **поточковим множенням** у частотній області — замість дорогої просторової згортки.

---

## Спільний setup (перша клітинка)

```python
def dft2(img):
    return np.fft.fftshift(np.fft.fft2(img.astype(np.float32)))

def idft2(F):
    return np.fft.ifft2(np.fft.ifftshift(F)).real.astype(np.float32)

def log_amp(F):
    return np.log1p(np.abs(F)).astype(np.float32)

def norm(arr):
    mn, mx = arr.min(), arr.max()
    return (arr - mn) / (mx - mn + 1e-8)
```

- `dft2` — обчислює ДПФ і зміщує DC у центр (`fftshift`)
- `idft2` — повертає DC на місце (`ifftshift`) і виконує зворотне ДПФ; беремо `.real`, бо уявна частина — лише числовий шум
- `log_amp` — логарифмічна амплітуда: $\log(1 + |F|)$. Потрібна тому, що DC-компонента на кілька порядків більша за решту — без логарифму всі деталі зникають
- `norm` — нормалізація до $[0, 1]$ для коректного відображення через `imshow`

Зображення завантажуються у відтінках сірого і конвертуються у `float32` — це важливо, бо ДПФ працює з дійсними числами, а не з uint8.

### Що робиться
```python
F = Fs[i]
row[0].imshow(norm(img), cmap="gray")
row[1].imshow(norm(log_amp(F)), cmap="inferno")
row[2].imshow(np.angle(F), cmap="hsv", vmin=-np.pi, vmax=np.pi)
```

Для кожного зображення:
1. Обчислюється ДПФ (`Fs[i]` вже підготовлено у setup)
2. Візуалізується логарифм амплітуди — `log_amp(F)` = $\log(1 + |F[k,l]|)$
3. Візуалізується фаза — `np.angle(F)` = $\arg(F[k,l]) \in [-\pi, \pi]$

### Що досягається
- **Амплітудна карта** показує, які частоти домінують. DC (центр після fftshift) — найяскравіший пік, бо відповідає середній яскравості. Горизонтальні/вертикальні смуги у спектрі — краї вздовж відповідних напрямків.
- **Фазова карта** виглядає майже випадково — але саме вона кодує **структуру і розташування** об'єктів. Це стане очевидним у завданнях 2 і 3.

## Завдання 2 — Заміна фаз між зображеннями

### Мета
Дослідити, яку роль відіграє фаза: реконструювати зображення з **власною амплітудою**, але **фазою від іншого зображення** — і побачити, чиї структури домінують у результаті.

### Математика
Кожне комплексне значення ДПФ можна записати у полярній формі:

$$F[k,l] = |F[k,l]| \cdot e^{j \angle F[k,l]}$$

Заміна фази — це побудова нового спектра:

$$F_{\text{new}}[k,l] = |F_i[k,l]| \cdot e^{j \angle F_j[k,l]}$$

де $i$ — власне зображення, $j$ — донор фази.

### Що робиться
```python
j = (i + 1) % n
recon = idft2(np.abs(Fs[i]) * np.exp(1j * np.angle(Fs[j])))
```

`np.abs(Fs[i])` — амплітуда власного зображення  
`np.angle(Fs[j])` — фаза сусіднього зображення  
`np.exp(1j * ...)` — перетворення фази назад у комплексну експоненту  
`idft2(...)` — зворотне ДПФ для отримання просторового результату

### Що досягається
Реконструйоване зображення **нагадує донора фази**, хоча амплітуда взята від оригіналу. Це доводить, що **фаза несе структурну інформацію** (краї, форми), тоді як амплітуда — лише "скільки" кожної частоти присутнє, але не "де".

---

## Завдання 3 — Випадкові фази

### Мета
Зруйнувати структуру зображення, зберігши амплітудний спектр — щоб переконатися, що **без правильної фази відновити зображення неможливо**.

### Що робиться
```python
rand_P = rng.uniform(-np.pi, np.pi, img.shape)
recon = idft2(np.abs(Fs[i]) * np.exp(1j * rand_P))
```

Фаза замінюється рівномірно розподіленими випадковими значеннями з $[-\pi, \pi]$. Амплітуда — незмінна.

### Що досягається
Результат — **шум**, що виглядає як текстура без будь-якої впізнаваної структури. Попри те, що "кількість" кожної частоти точно така ж, як в оригіналі, зображення абсолютно нечитабельне. Це підтверджує: **фаза критична** для перцепції форм і структур.

## Завдання 4a — Константна амплітуда 1/DC

### Мета
Замінити всю амплітуду на одне константне значення і переконатися, що структура зображення **зберігається лише завдяки фазі**.

### Математика
DC-компонента ДПФ:

$$F\left[\frac{M}{2}, \frac{N}{2}\right] = \sum_{m,n} f[m,n]$$

Це сума всіх пікселів зображення — пропорційна до середньої яскравості. Ми встановлюємо амплітуду всіх частот рівною $1 / |F_{\text{DC}}|$:

$$F_{\text{new}}[k,l] = \frac{1}{|F_{\text{DC}}|} \cdot e^{j \angle F[k,l]}$$

### Що робиться
```python
dc = np.abs(F[h // 2, w // 2])
recon = idft2((1.0 / max(dc, 1e-6)) * np.exp(1j * np.angle(F)))
```

`F[h//2, w//2]` — центральний елемент після fftshift — і є DC.  
Константна амплітуда `1/dc` множиться на комплексну експоненту оригінальної фази.

### Що досягається
Реконструкція демонструє **контури і структуру** зображення — чітко впізнавані, хоча глобальна яскравість "вирівняна". Різні частоти мають однаковий внесок за амплітудою, тому зображення нагадує **edge-map**: низькочастотні деталі (рівномірні зони) зникають, але краї залишаються.

---

## Завдання 4b — Занулення квадранта амплітуди

### Мета
Дослідити, як **асиметричне видалення частини амплітудного спектра** впливає на просторове зображення.

### Математика
ДПФ реального зображення має властивість **спряженої симетрії**:

$$F[k,l] = F^*[M-k, N-l]$$

Тобто спектр симетричний відносно центру. Занулення одного квадранта порушує цю симетрію, що вводить **уявну складову** у реконструкцію — тому результат може мати артефакти. Беремо `.real` через `idft2`, отже уявна частина відкидається.

### Що робиться
```python
def zero_quadrant(A, q):
    slices = [...]
    A[slices[q]] = 0
    return A

A_mod = zero_quadrant(np.abs(F), q)
recon = idft2(A_mod * np.exp(1j * np.angle(F)))
```

Обирається випадковий квадрант (0=верхній лівий, 1=верхній правий, 2=нижній лівий, 3=нижній правий) і всі значення амплітуди в ньому замінюються нулями. Фаза — незмінна.

### Що досягається
Реконструкція втрачає частину просторових частот — відповідно до геометрії спектра. Занулення квадранта несиметричне (оскільки кожен квадрант у спектрі пов'язаний із напрямком частот у просторі), тому зображення стає **несиметрично розмитим або спотвореним** у певних напрямках.

## Завдання 1 — Класичні фільтри через ДПФ

### Мета
Застосувати стандартні фільтри обробки зображень (Sobel, Gaussian, Mean, Laplacian, Scharr) **виключно через ДПФ**, не використовуючи пряму просторову згортку — і водночас візуалізувати амплітудні спектри самих ядер.

### Математика (теорема про згортку)
Якщо $g = f * h$, то:

$$G = \mathcal{F}\{f\} \cdot \mathcal{F}\{h\} \implies g = \mathcal{F}^{-1}\{F \cdot H\}$$

Тобто замість обчислення згортки можна:
1. Взяти ДПФ зображення і ядра
2. Помножити їх поточково
3. Застосувати ЗЗДПФ

### Ключова тонкість: `pad_kernel`
```python
def pad_kernel(kernel, size):
    canvas = np.zeros((size, size), np.float32)
    canvas[:h, :w] = kernel
    canvas = np.roll(canvas, -(h // 2), axis=0)
    canvas = np.roll(canvas, -(w // 2), axis=1)
    return canvas
```

Просто розміщення ядра у лівому верхньому куті дасть **зсув фази** у результаті (еквівалент некентрованої згортки). `np.roll` циклічно зміщує ядро так, що його **центр опиняється на позиції [0,0]** — тобто DC-позиції для `fft2` (без fftshift). Це усуває фазовий зсув у результаті.

```python
def apply_filter_fft(img, kernel):
    return idft2(dft2(img) * kernel_to_freq(kernel, img.shape[0]))
```

`kernel_to_freq` — це `fftshift(fft2(pad_kernel(...)))`, тобто shifted спектр ядра. Множення відбувається в shifted координатах, де обидва спектри вирівняні по центру DC.

### Ядра і їх властивості

| Фільтр | Ядро | Тип у частотній області |
|--------|------|------------------------|
| Sobel V | $[[-1,0,1],[-2,0,2],[-1,0,1]]$ | Посилює вертикальні краї — high-pass по горизонталі |
| Sobel H | транспонований Sobel V | Горизонтальні краї |
| Gaussian | $g(x,y) = e^{-(x^2+y^2)/2\sigma^2}$, нормований | Low-pass — розмивання |
| Mean | $\frac{1}{25}\mathbf{1}_{5\times5}$ | Low-pass — рівномірне усереднення |
| Laplacian | $[[0,1,0],[1,-4,1],[0,1,0]]$ | High-pass — виділяє локальні мінімуми і максимуми |
| Scharr V/H | посилена версія Sobel з ваговими коефіцієнтами 3/10 | Edge detector з кращою ізотропністю |

Візуалізація `log|DFT ядра|` підтверджує теоретичні очікування: Gaussian у спектрі — це знову Gaussian (гауссівський low-pass), Sobel — це high-pass по одній осі.

## Завдання 2 — Власні частотні фільтри: low-pass, high-pass, band-pass

### Мета
Реалізувати прості фільтри безпосередньо у частотній області через **кільцеві маски** — без будь-якого ядра у просторі.

### Математика
Після `fftshift` відстань від центру масиву пропорційна частоті:
- Центр ($r = 0$) — DC, нульова частота
- Край ($r \approx 1$) — максимальна просторова частота

Нормована радіальна відстань:
```python
def radial_distance(size):
    r = np.hypot(x - cx, y - cy) / (size / 2)
```

На основі цієї відстані будуються маски:

```python
# Low-pass: пропустити лише низькі частоти (r ≤ cutoff)
(radial_distance(SIZE) <= 0.10).astype(np.float32)

# High-pass: пропустити лише високі частоти (r > cutoff)
(radial_distance(SIZE) > 0.25).astype(np.float32)

# Band-pass: пропустити частоти у кільці [low, high]
((r >= 0.10) & (r <= 0.30)).astype(np.float32)
```

Маска — це бінарний масив того ж розміру, що й зображення. Множення спектра на маску **обнуляє заборонені частоти**.

### Що досягається
- **LPF** — розмиває зображення (зберігає лише грубі форми)
- **HPF** — залишає лише краї і дрібні деталі (висока просторова частота = швидкі зміни яскравості)
- **BPF** — виділяє певний масштаб деталей (наприклад, середні текстури)

## Завдання 4 — Просторові ядра через зворотне ДПФ маски

### Мета
З'ясувати, яке **просторове ядро згортки** відповідає нашим частотним маскам — тобто отримати відповідь на питання "якщо цей ж ефект реалізувати через пряму згортку, яке ядро треба використовувати?"

### Математика
По теоремі про згортку: якщо маска $H[k,l]$ у частотній області — це передатна функція фільтра, то відповідне просторове ядро:

$$h[m,n] = \mathcal{F}^{-1}\{H[k,l]\}$$

```python
def mask_to_kernel(mask):
    return np.fft.ifftshift(np.fft.ifft2(np.fft.ifftshift(mask))).real
```

`ifftshift` перед `ifft2` — прибирає shift перед зворотним перетворенням.  
`ifftshift` після `ifft2` — центрує ядро (DC у центрі просторового масиву).

### Що досягається
Для LPF ядро виглядає як **sinc-функція** (функція Бесселя у 2D) — математично очікуваний результат: прямокутна маска у частотній → sinc у просторі. Для BPF — різниця двох sinc. Ядра займають весь масив `SIZE×SIZE`, тому для візуалізації вирізається центральний кроп `CROP×CROP`.

## Завдання 5 — Порівняння DFT-фільтрації зі звичайною згорткою

### Мета
Перевірити теорему про згортку **кількісно**: переконатися, що фільтрація через ДПФ дає той самий результат, що й пряма просторова згортка з похідним ядром.

### Що робиться
```python
dft_out  = dft_results[fname][0]
conv_out = cv2.filter2D(img0, cv2.CV_32F, h_full, borderType=cv2.BORDER_WRAP)
diff     = np.abs(dft_out - conv_out)
rmse     = np.sqrt(np.mean(diff ** 2))
```

`cv2.filter2D` виконує **пряму просторову згортку** з ядром `h_full` — повним масивом `256×256`.  
`borderType=cv2.BORDER_WRAP` — циклічне доповнення країв, що відповідає припущенням ДПФ (ДПФ за замовчуванням вважає сигнал **циклічним**).

### Що досягається
RMSE між двома методами близьке до нуля (зазвичай < 0.01 у масштабі 0–255) — що підтверджує **еквівалентність** фільтрації через ДПФ і прямої згортки для лінійних фільтрів. Залишкова різниця — лише числова похибка floating-point.

# Блок 3 — Редагування зображень у частотній області

---

## Завдання 1 — Вибір зображення з виразними піками в амплітуді

### Мета
Знайти зображення, де у ДПФ-амплітуді є **чіткі локальні піки** (яскраві точки), що відповідають конкретним просторовим структурам — наприклад, регулярним текстурам, ґраткам, смугам.

### Чому виникають піки?
Регулярний патерн у просторі — наприклад, ґратка з кроком $T$ пікселів — у частотній області дає **дискретний спектр**: енергія зосереджена у частотах $f = 1/T$ і її гармоніках. На амплітудній карті це виглядає як яскраві точки, симетрично розташовані відносно DC.

```python
def show_spectrum(idx):
    axes[1].imshow(norm(log_amp(F)), cmap="inferno")
```

Інтерактивний dropdown дозволяє переглядати спектри всіх зображень і вибрати найкраще для редагування.

## Завдання 2 — Занулення амплітуди у точках піків

### Мета
**Видалити** з зображення регулярну структуру, занулюючи відповідні частотні компоненти.

### Математика — спряжена симетрія
ДПФ реального зображення завжди має властивість:

$$F[k, l] = F^*[M-k \pmod{M},\ N-l \pmod{N}]$$

Тому кожному піку відповідає **симетричний партнер** на протилежному боці від DC. Обидва треба обнулити — інакше результат не буде реальним числом (виникне уявна складова).

```python
def suppress_notch(F, rc, radius=5):
    r2, c2 = (h - r) % h, (w - c) % w
    for pr, pc in [(r, c), (r2, c2)]:
        F[(y - pr)**2 + (x - pc)**2 <= radius**2] = 0
```

Кругова маска (notch filter) радіуса `radius` пікселів обнуляє обидва симетричних піки.

### Автоматичне виявлення піків
```python
def find_spikes(F, top_k=6, dc_radius=10):
    amp[np.hypot(y - h//2, x - w//2) < dc_radius] = 0   # ігноруємо DC
    for _ in range(top_k):
        idx = np.argmax(amp)
        r, c = divmod(int(idx), w)
        peaks.append((r, c))
        amp[max(0,r-5):r+6, max(0,c-5):c+6] = 0   # прибираємо вже знайдений пік
```

Ітеративно знаходимо найяскравіші точки (greedy search), кожного разу маскуючи вже знайдений пік, щоб не вибирати його повторно.

## Завдання 3 — Перевірка результату через зворотне ДПФ

### Мета
Переконатися, що після занулення піків **відповідна структура зникає** з просторового зображення.

### Що робиться
```python
recon = idft2(F_edited)
```

Зворотне ДПФ відредагованого спектра дає просторове зображення. Порівнюється з оригіналом.

```python
print(f"RMSE: {np.sqrt(np.mean((img_work - recon)**2)):.3f}")
print(f"Fraction zeroed: {(F_edited == 0).mean()*100:.2f}%")
```

- **RMSE** — кількісна різниця між оригіналом і реконструкцією. Має бути ненульовою, але невеликою (занулені лише окремі частоти)
- **Fraction zeroed** — яка частка спектра обнулена. Зазвичай < 1% — але ефект значний, бо видалено найбільш енергетично потужні компоненти

### Що досягається
Метод відомий як **notch filtering** — вузькосмугове придушення конкретних частот. Він використовується для:
- Видалення муару (moire patterns) зі сканованих зображень
- Придушення регулярних артефактів (наприклад, пікселізації від стиснення)
- Видалення регулярних перешкод у медичній візуалізації (наприклад, ґраток у рентгені)

Інтерактивні слайдери дозволяють в реальному часі підбирати `top_k` (кількість пар піків) і `radius` (розмір notch) — щоб досягти оптимального балансу між видаленням артефакту і збереженням деталей зображення.
